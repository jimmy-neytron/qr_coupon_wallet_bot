# QR Coupon Wallet — Telegram Mini App

QR Coupon Wallet — это Telegram Mini App для хранения QR-купонов и текстовых промокодов.

Приложение позволяет сохранять промокоды лично или в общей коллекции, распределять их по группам, открывать QR-коды повторно и пользоваться уже сохранёнными данными даже без интернета.

Проект разработан как frontend-приложение на Vue 3 с использованием Supabase в качестве backend-части.

---

## Возможности

В приложении реализованы:

- сохранение QR-кодов и текстовых промокодов;
- генерация QR-кода из сохранённой строки;
- сканирование QR через Telegram Mini App API;
- личные и общие коллекции промокодов;
- приглашение другого пользователя в общую коллекцию по коду;
- группы промокодов;
- избранные промокоды;
- архивирование промокодов;
- срок действия промокода;
- поиск и фильтрация;
- offline-first режим;
- локальное хранение данных в IndexedDB;
- очередь синхронизации при отсутствии интернета;
- автоматическая синхронизация после восстановления сети;
- toast-уведомления;
- адаптивный интерфейс под Telegram WebView;
- защита от случайного сворачивания Telegram Mini App при скролле;
- unit-тесты с покрытием более 90%.

---

## Основной сценарий работы

Пользователь открывает Mini App в Telegram, сканирует QR-код купона или добавляет промокод вручную.

После сохранения промокод попадает в выбранную коллекцию и группу.

При необходимости пользователь открывает сохранённый промокод, а приложение заново генерирует QR-код из сохранённой строки.

Если интернет пропадает, приложение продолжает работать с локальной копией данных. Все изменения сохраняются в IndexedDB и отправляются в Supabase после восстановления соединения.

## Архитектура данных

В основе приложения лежит модель:

```txt
User → Collection → Group → Coupon
```

Где:

- `User` — пользователь Telegram;
- `Collection` — личная или общая коллекция;
- `Group` — группа промокодов внутри коллекции;
- `Coupon` — QR-купон или текстовый промокод.

Такая структура позволяет хранить как личные данные, так и общие промокоды для нескольких пользователей.

## Offline-first режим

Приложение поддерживает работу без интернета.

Для этого используется:

- IndexedDB для локального хранения данных;
- Service Worker для кеширования интерфейса;
- sync queue для отложенной синхронизации.

Когда интернет недоступен, приложение позволяет:

- просматривать уже загруженные промокоды;
- открывать QR-коды;
- добавлять промокоды вручную;
- создавать группы;
- редактировать и удалять локальные данные;
- архивировать и добавлять в избранное.

После восстановления соединения изменения автоматически отправляются в Supabase.

## Telegram Mini App

Приложение использует Telegram Mini App API:

- `window.Telegram.WebApp`;
- `initData`;
- `showScanQrPopup`;
- `HapticFeedback`;
- `disableVerticalSwipes`.

`initData` отправляется на backend и проверяется в Supabase Edge Function.

Это позволяет определять Telegram-пользователя и защищать доступ к коллекциям.

## Особенности реализации

### Offline fallback

Если приложение считает, что интернет есть, но запрос к серверу падает из-за сетевой ошибки, действие автоматически сохраняется локально.

Это позволяет избежать ситуации, когда пользователь нажимает кнопку, получает ошибку, а только повторное действие сохраняется офлайн.

### Sync queue

Все офлайн-операции попадают в очередь:

```txt
create
update
delete
```

Когда интернет появляется, очередь последовательно отправляется на сервер.

После успешной синхронизации локальные id заменяются на серверные.

### Duplicate handling

При сохранении промокода приложение проверяет дубли внутри активной коллекции.

Если такой QR-код или промокод уже существует, пользователь получает понятное сообщение.

### Telegram WebView UX

Для удобства внутри Telegram Mini App реализованы:

- фиксированная нижняя навигация;
- внутренний скролл приложения;
- защита от вертикальных свайпов, которые могут свернуть Mini App;
- нижние toast-уведомления;
- haptic feedback на успешные и ошибочные действия.
